# 页面模板索引

生成页面前，必须先判断页面类型，再读取对应模板。页面模板的约束优先级高于通用组件参数。

## 默认生成模式（强约束）

生成 **B 端后台页面** 时，**默认必须带应用壳**，不得默认输出无菜单的独立整页，除非用户明确要求「仅内容页」「不要菜单」「独立 HTML 页面」。

| 默认行为 | 要求 |
| ---- | ---- |
| 输出形态 | `Layout` + 左侧 `Sider` + `Menu` + 顶栏 `Header` + `Content` 业务区 |
| 必读模板 | **始终先读** `app-shell.md`，再读业务页模板（如 `list-table.md`） |
| 页面注册 | 新业务页必须写入 `PAGE_REGISTRY` 并挂到菜单，支持点击切换 |
| HTML 原型 | 优先基于 `admin-shell.html` 扩展；或单文件内 `PAGE_REGISTRY` + 多业务组件 |
| 业务页范围 | 业务组件只渲染 `Content` 内部，不重复 `Sider` / 全局 `Layout` |

**仅当用户明确说**「不要左侧菜单」「只要页面内容」「单页原型」时，才可跳过 `app-shell.md`，直接按 `list-table.md` 等生成独立页（如 `站点映射.html`）。

## 页面类型索引

| 页面类型 | 触发描述 | 必读模板 |
| ---- | ---- | ---- |
| 后台应用壳（**默认必有**） | 后台系统、管理后台、列表页、详情页、B 端页面（未特别声明独立页时） | `app-shell.md` |
| 查询表格页 | 列表页、查询表格、表格+查询、管理列表、TableList | `list-table.md` |
| 详情页 | 详情页、基础详情、查看详情、Profile、Basic Profile | `detail-basic.md` |
| 表单弹窗 / 基础表单 | 新增弹窗、编辑弹窗、表单弹窗、基础表单、Basic Form、Modal 表单 | `form-modal.md` |

## 读取规则

1. 先读取根目录 `ant-design-system-prompt.md`。
2. **默认按「后台应用壳 + 业务页」模式**：先读 `app-shell.md`，再读业务模板。
3. 根据用户需求判断业务页类型（TableList / 详情 / 弹窗表单等），读取对应模板。
4. 若用户明确要求「仅内容页 / 无菜单」，才跳过 `app-shell.md`。
5. 若页面同时包含多种形态，按页面主体类型优先，再叠加局部组件规则。
6. 如果无法判断页面类型，先向用户确认，不得自由发挥布局。
